# Wager run, 2026-06-22

The wager test, run for the first time. Embodied prompt v4 against the flat control, on a fixed set of felt-threshold reflections, blind-graded by independent agents who did not write either prompt and never saw which condition was which.

The question (from `frontier-lab/the-story.md`): does an embodied frame read a person's thresholds that a flat reflective prompt misses.

## Verdict: null

On this set the body map did not earn its place. By item majority the two prompts tied 5 to 5. By raw blind vote the embodied prompt led 16 to 14 of 30, which is well inside noise for a set this size. There is no detectable advantage for the embodied frame here.

The finding under the null is the useful part. The variable that actually predicted a good response was not embodiment. It was restraint. Whichever response stayed leaner and met the person's state without piling on won, with or without the body map. The embodied prompt's characteristic failure is over-functioning: it deploys a soma read, a compass heading, a body-location question, or a practice when the state called for it to stay out of the way. The map gives the model more to say, and more to say is usually the wrong move for an activated or flat nervous system.

## Method

- **Conditions.** Embodied = `embodied-service/worker/src/prompt.js` (embodied-prompt-v4) verbatim. Flat = `embodied-service/verify/flat-prompt.js` verbatim. Same boundaries, same voice, the only difference is the body map.
- **Inputs.** 12 felt-threshold reflections: the 6 placeholders from `verify/wager.mjs` plus 6 composed in the same shape, spanning the co-regulation states the prompt claims to read (scattered, activated, flat, flow, somatic drop, circling, anxious wake, quiet pride, overwhelm, held breath, grief). This is a designed, public-safe set, not Isaac's real entries (see limits).
- **Generation.** For each reflection, one agent emulated the embodied prompt and one the flat prompt, reading the verbatim text from disk. Both responses for an item were generated the same way, so the A/B comparison is fair.
- **Blind grading.** Each item's two responses were labeled A/B with the embodied side alternated by item to balance position. Three independent grader agents per item judged which response better read the threshold and met the person, blind to which was which and to the key. The rubric explicitly told graders not to reward body-part names, jargon, or length, and that a response that names anatomy but misses the person loses.
- **Tally.** Deterministic, after grading, by revealing the per-item key.

## Honest limits

This run is real but it is not the run the PROPOSAL calls Phase A. What it is not:

1. **Not Isaac's own reflections.** The set is designed and public-safe because the alchemy repo is public and his real Lived Record entries should not be committed here. `verify/wager.mjs` is blunt that placeholder inputs "only prove the harness works." The definitive run on his real entries is still owed and should be vault-private.
2. **Not the pinned harness.** Generation emulated the prompts via agents rather than the SEL-1 service's pinned call (claude-opus-4-8, max_tokens 1024, adaptive thinking). Same model family, fair within the comparison, but not the deployed path.
3. **Not a human grader.** Graders were independent Claude subagents, blind to condition and key. That is independent of the prompt author, which is the bar `the-story.md` sets, but it is not a human reader. The independent live human eval is still owed.
4. **Two items excluded.** On items 7 and 9 a generator produced a meta-refusal about a missing file instead of staying in character (the read-from-disk emulation failed). Both were excluded. They happened to cut in opposite directions (one spurious embodied win, one spurious flat win), so excluding them does not move the result.
5. **Small N.** 10 clean items, 3 graders each. A 16 to 14 split is noise. Read this as "no advantage detected," not "flat is better."

## The pattern under the null

Read down the "why the loser lost" column. The losing move is almost always the same move, and either prompt can make it: doing too much. Adding a question, a frame, a practice, a recap, a health flag, a piece of interpretation, when the person's state asked to be met and left alone.

The embodied prompt makes this mistake more often because the body map hands it more to deploy. When the map stayed in the model's read and did not surface as output, the embodied prompt won (items 0, 2, 5, 8, 11: it refused to weld, got out of the way, de-pressurized, grounded, offered one lean somatic invitation). When the map surfaced as a soma read, a compass heading, a body-location question, or a practice, it lost (items 1, 4, 6).

That points at a concrete next prompt, an embodied-prompt-v5 hypothesis: keep the body map as a private reading aid and forbid it from surfacing as output unless the person reaches for it first. Read with the map, respond with restraint. The wager would then be re-run to test whether a map-as-silent-read beats both the surfacing version and the flat control.

## What this run settles and what it does not

Settles: the body map does not earn its place by being spoken. A v4 that surfaces its apparatus loses to a plain reflective companion about as often as it wins.

Does not settle: whether the map earns its place as a silent read (the v5 hypothesis), and whether any of this holds on Isaac's real entries graded by a human. Those are the Phase A run the PROPOSAL still asks for.
